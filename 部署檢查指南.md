# 🔍 GitHub Pages 部署檢查指南

## ✅ 檢查步驟

### 1. 檢查 GitHub Actions 狀態

**訪問：** https://github.com/dropout-tech/playtt/actions

**應該看到：**
- ✅ "Deploy to GitHub Pages" workflow
- ✅ 狀態應該是 "綠色勾勾"（成功）或 "黃色圓圈"（進行中）

**如果看到：**
- ❌ **紅色叉叉**：構建失敗，點擊查看錯誤日誌
- ⏳ **黃色圓圈**：正在構建，請等待
- ✅ **綠色勾勾**：構建成功，部署完成

### 2. 檢查網站是否可以訪問

**訪問：** https://dropout-tech.github.io/playtt

**應該看到：**
- ✅ 網站正常顯示
- ✅ 首頁 Banner 正常
- ✅ 響應式設計正常

**如果看到：**
- ❌ **404 Not Found**：Pages 可能還沒部署完成，等待幾分鐘
- ❌ **空白頁面**：檢查瀏覽器控制台錯誤
- ⏳ **正在加載**：正常，等待載入完成

### 3. 檢查 Pages 設置

**訪問：** https://github.com/dropout-tech/playtt/settings/pages

**確認：**
- ✅ Source: **GitHub Actions**
- ✅ 顯示部署狀態和 URL

### 4. 檢查構建日誌

**訪問 Actions 頁面，點擊最新的 workflow 運行：**

**應該看到：**
1. ✅ **Checkout** - 成功
2. ✅ **Setup Node.js** - 成功
3. ✅ **Install dependencies** - 成功
4. ✅ **Build** - 成功
5. ✅ **Setup Pages** - 成功
6. ✅ **Upload artifact** - 成功
7. ✅ **Deploy to GitHub Pages** - 成功

**如果有錯誤：**
- 點擊失敗的步驟查看詳細日誌
- 檢查錯誤訊息

## 🔧 常見問題排查

### 問題 1：Actions 沒有自動觸發

**解決：**
1. 確認已推送代碼到 `master` 分支
2. 確認 `.github/workflows/deploy.yml` 文件存在
3. 手動觸發：Actions → Deploy to GitHub Pages → Run workflow

### 問題 2：構建失敗

**常見原因：**
- Node.js 版本不兼容
- 依賴安裝失敗
- 構建錯誤

**解決：**
1. 查看構建日誌
2. 檢查 `package.json` 配置
3. 確認所有依賴正確

### 問題 3：網站 404

**可能原因：**
- Pages 還沒部署完成（等待 5-10 分鐘）
- Source 設置錯誤
- 構建失敗

**解決：**
1. 等待幾分鐘
2. 確認 Pages Source 設置為 "GitHub Actions"
3. 檢查 Actions 是否成功

### 問題 4：網站空白或錯誤

**檢查：**
1. 瀏覽器控制台（F12）查看錯誤
2. 檢查構建是否成功
3. 確認 `package.json` 中的 `homepage` 字段正確
